## 1. 設計方針

### 基本原則

- **既存 kaedevn DB と同一 PostgreSQL インスタンスに追加** — User テーブルを FK で共有し、認証を一元化
- **MMO 専用テーブルには `mmo_` プレフィックス** — 既存テーブル（`users`, `projects`, `works` 等）との名前衝突を回避
- **マスターデータとプレイヤーデータの分離** — `*Master` テーブル（ゲーム定義）と `*` テーブル（プレイヤー状態）を明確に分ける
- **サーバー権威モデル** — Colyseus サーバーが状態を管理し、DB は永続化レイヤー。リアルタイム状態は Redis / Colyseus Room State で保持
- **BigInt タイムスタンプ** — kaedevn 既存スキーマに合わせて `BigInt`（Unix ms）を使用
- **ID 形式** — kaedevn に合わせて `VarChar(31)` の CUID/ULID 系 ID

### Redis との役割分担

| データ | PostgreSQL | Redis |
|--------|-----------|-------|
| プレイヤーキャラクター（永続） | PlayerCharacter | - |
| インベントリ（永続） | Inventory | - |
| クエスト進行（永続） | QuestProgress | - |
| チャットログ（永続） | ChatLog | 最新100件キャッシュ |
| リアルタイム位置 | - | Colyseus Room State |
| オンライン状態 | - | Colyseus Presence |
| セッション | - | Redis Session Store |
| ランキング | - | Redis Sorted Set |

## 2. 既存テーブルとの関係

### 共有する既存テーブル

| 既存テーブル | MMO での使用方法 |
|-------------|----------------|
| **User** (`users`) | MMO プレイヤーのアカウント。`PlayerCharacter.userId` で FK 参照 |
| **Wallet** (`wallets`) | MMO 内課金（プレミアム機能、AI NPC 対話等）にも流用可能 |
| **PointTransaction** (`point_transactions`) | MMO 課金のトランザクション記録 |

### 使用しないテーブル

Project, Work, Asset, Character 等のビジュアルノベル制作系テーブルは MMO では直接参照しない。MMO のゲームデータは専用のマスターテーブルで管理する。

  @@index([userId, isActive])

// TradeLog — 取引履歴
model TradeLog {

  buyer  PlayerCharacter @relation("TradeBuyer", fields: [buyerId], references: [id], onDelete: Cascade)
  seller PlayerCharacter @relation("TradeSeller", fields: [sellerId], references: [id], onDelete: Cascade)
  item   ItemMaster      @relation(fields: [itemId], references: [id], onDelete: Restrict)

  @@index([buyerId, createdAt(sort: Desc)])
  @@index([sellerId, createdAt(sort: Desc)])
  @@index([itemId])
  @@index([type, createdAt(sort: Desc)])
  @@map("mmo_trade_logs")
}

// BattleLog — 戦闘履歴
model BattleLog {

  goldGained  Int     @default(0) @map("gold_gained")

  battleData  Json?   @map("battle_data") // { "enemies": [...], "actions": [...], "log": "..." }
  durationMs  Int     @default(0) @map("duration_ms")
  createdAt   BigInt  @map("created_at")

  character PlayerCharacter @relation(fields: [characterId], references: [id], onDelete: Cascade)

  @@index([characterId, createdAt(sort: Desc)])
  @@index([battleType, createdAt(sort: Desc)])

## 4. テーブル詳細

### 4.1 PlayerCharacter（プレイヤーキャラクター）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | VarChar(31) | PK (CUID) |
| userId | VarChar(31) | FK → User.id |
| name | VarChar(50) | キャラクター名（ユーザーごとにユニーク） |
| gender | VarChar(10) | "female" / "male" |
| preset | VarChar(30) | 図形キャラプリセット（hanako, taro 等） |
| level | Int | 現在レベル |
| exp | Int | 累計経験値 |
| hp, maxHp | Int | 現在HP / 最大HP |
| mp, maxMp | Int | 現在MP / 最大MP |
| attack, defense, speed | Int | 基礎ステータス |
| gold | Int | 所持ゴールド |
| zoneId | VarChar(50) | 現在のゾーンID |
| x, y | Int | ゾーン内の位置座標 |
| expression | VarChar(30) | 最後の表情（18種のいずれか） |
| pose | VarChar(30) | 最後のポーズ（16種のいずれか） |
| isActive | Boolean | 現在使用中のキャラか |
| totalPlayMs | BigInt | 累計プレイ時間（ミリ秒） |
| lastLoginAt | BigInt | 最終ログイン日時 |
| createdAt | BigInt | 作成日時 |
| updatedAt | BigInt | 更新日時 |

**永続化タイミング:**
- `onJoin`: DB からロード → Colyseus Room State に反映
- `onLeave`: Room State → DB に保存
- `onDispose`: 全プレイヤーを DB に保存
- 定期保存: 5分ごとにダーティフラグのあるプレイヤーを保存

### 4.2 ItemMaster（アイテム定義）

| id | VarChar(31) | PK |
| slug | VarChar(50) | ユニーク識別子 |
| name | VarChar(100) | 表示名 |
| type | VarChar(20) | consumable / equipment / key / material |
| subtype | VarChar(20) | weapon / armor / accessory（equipment のみ） |
| rarity | VarChar(20) | common / uncommon / rare / epic / legendary |
| maxStack | Int | 最大スタック数 |
| buyPrice, sellPrice | Int | NPC売買価格 |
| *Bonus | Int | 装備時のステータス修正（hp, mp, atk, def, spd） |
| effect | Json | 消費時の効果定義 |

### 4.3 Inventory（インベントリ）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | VarChar(31) | PK |

| itemId | VarChar(31) | FK → ItemMaster.id |
| quantity | Int | 所持数 |
| isEquipped | Boolean | 装備中か |
| slot | VarChar(20) | 装備スロット（weapon / armor / accessory） |

### 4.6 ZoneMaster（ゾーン定義）

| カラム | 型 | 説明 |
|--------|-----|------|
| id | VarChar(50) | PK（slug 形式: "zone-001-village"） |
| name | VarChar(100) | ゾーン名 |
| type | VarChar(20) | field / town / dungeon / safe |
| maxPlayers | Int | 最大収容人数（Colyseus Room の maxClients） |
| level | Int | 推奨レベル |
| bgAssetId | VarChar(100) | kaedevn 背景アセットID |
| bgmSlug | VarChar(50) | BGM スラグ |
| adjacentZones | Json | 隣接ゾーンリスト |
