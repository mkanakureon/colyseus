## 1. Client クラス（ColyseusSDK）

**ファイル:** `packages/sdk/src/Client.ts`

SDK のエントリーポイント。サーバーへの接続設定を保持し、Room のマッチメイキングを行う。

> `Client` は `ColyseusSDK` クラスのエイリアスとしてエクスポートされている。

### 型パラメータ

| パラメータ | 説明 |
|-----------|------|
| `ServerType extends SDKTypes` | サーバー側の型定義（Room 名・ルート定義を推論） |
| `UserData` | 認証ユーザーデータの型 |

### コンストラクタ

```typescript
constructor(
    settings: string | EndpointSettings = DEFAULT_ENDPOINT,
    options?: ClientOptions
)
```

**引数:**

| 引数 | 型 | 説明 |
|------|-----|------|
| `settings` | `string \| EndpointSettings` | サーバー URL または接続設定オブジェクト。省略時は `window.location` から自動推論、またはフォールバックで `ws://127.0.0.1:2567` |
| `options` | `ClientOptions` | カスタムヘッダー、URL ビルダー、プロトコル指定 |

**`EndpointSettings` インターフェース:**

```typescript
interface EndpointSettings {
    hostname: string;
    secure: boolean;
    port?: number;
    pathname?: string;
    searchParams?: string;
    protocol?: "ws" | "h3";
}
```

**`ClientOptions` インターフェース:**

```typescript
interface ClientOptions {
    headers?: { [id: string]: string };
    urlBuilder?: (url: URL) => string;
    protocol?: "ws" | "h3";
}
```

### プロパティ

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `http` | `HTTP<ServerType['~routes']>` | HTTP リクエスト用クライアント |
| `auth` | `Auth<UserData>` | 認証モジュール |
| `settings` | `EndpointSettings` | 接続設定 |

### 静的プロパティ

| プロパティ | 型 | 値 |
|-----------|-----|-----|
| `VERSION` | `string` | `"0.17"` |

### メソッド一覧

#### `joinOrCreate()`

Room が存在すれば参加、なければ新規作成する。最も一般的なマッチメイキングメソッド。

```typescript
public async joinOrCreate<State = any>(
    roomName: string,
    options?: JoinOptions,
    rootSchema?: SchemaConstructor<State>
): Promise<Room<any, State>>
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `roomName` | `string` | Room の名前 |
| `options` | `JoinOptions` | Room 参加時のオプション（サーバー側 `onJoin` の第2引数に渡される） |
| `rootSchema` | `SchemaConstructor<State>` | ルートスキーマクラス（省略可。クライアント側で型定義済みの場合に指定） |

**戻り値:** `Promise<Room<any, State>>`

---

#### `create()`

必ず新しい Room を作成する。

```typescript
public async create<State = any>(
    roomName: string,
    options?: JoinOptions,
    rootSchema?: SchemaConstructor<State>
): Promise<Room<any, State>>
```

引数は `joinOrCreate()` と同じ。

---

#### `join()`

既存の Room に参加する（空きがない場合はエラー）。

```typescript
public async join<State = any>(
    roomName: string,
    options?: JoinOptions,
    rootSchema?: SchemaConstructor<State>
): Promise<Room<any, State>>
```

引数は `joinOrCreate()` と同じ。

---

#### `joinById()`

Room ID を指定して直接参加する。

```typescript

    roomId: string,
    options?: JoinOptions,
    rootSchema?: SchemaConstructor<State>
): Promise<Room<any, State>>
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `roomId` | `string` | 参加する Room の ID |
| `options` | `JoinOptions` | 参加オプション |
| `rootSchema` | `SchemaConstructor<State>` | ルートスキーマクラス |

---

#### `reconnect()`

以前接続していた Room に再接続する。

```typescript
public async reconnect<State = any>(
    reconnectionToken: string,
    rootSchema?: SchemaConstructor<State>
): Promise<Room<any, State>>
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `reconnectionToken` | `string` | `room.reconnectionToken` から取得した再接続トークン。形式: `roomId:token` |
| `rootSchema` | `SchemaConstructor<State>` | ルートスキーマクラス |

> **注意:** 旧 API の `reconnect(roomId, sessionId)` 形式は廃止済み。`reconnectionToken` のみを受け付ける。

---

#### `consumeSeatReservation()`

サーバーから取得した座席予約情報を使って Room に接続する。高度なカスタムマッチメイキング向け。

```typescript

    response: ISeatReservation,
    rootSchema?: SchemaConstructor<T>

```

| 引数 | 型 | 説明 |
|------|-----|------|
| `response` | `ISeatReservation` | マッチメイキングレスポンス（`roomId`, `sessionId`, `name` 等を含む） |
| `rootSchema` | `SchemaConstructor<T>` | ルートスキーマクラス |

---

#### `getLatency()`

サーバーとの往復レイテンシを計測する。

```typescript
public getLatency(options?: LatencyOptions): Promise<number>
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `options` | `LatencyOptions` | 計測オプション |

**`LatencyOptions` インターフェース:**

```typescript
interface LatencyOptions {
    protocol?: "ws" | "h3";   // デフォルト: "ws"
    pingCount?: number;        // デフォルト: 1（複数回の場合は平均を返す）
}
```

**戻り値:** `Promise<number>` -- レイテンシ（ミリ秒）

---

#### `static selectByLatency()`

複数のエンドポイントから最もレイテンシが低いものを選択する。

```typescript
static async selectByLatency<ServerType extends SDKTypes = any, UserData = any>(
    endpoints: Array<string | EndpointSettings>,
    options?: ClientOptions,
    latencyOptions?: LatencyOptions
): Promise<ColyseusSDK<ServerType, UserData>>
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `endpoints` | `Array<string \| EndpointSettings>` | 候補となるエンドポイントの配列 |
| `options` | `ClientOptions` | クライアントオプション |
| `latencyOptions` | `LatencyOptions` | レイテンシ計測オプション |

**戻り値:** 最もレイテンシの低いエンドポイントで初期化された `ColyseusSDK` インスタンス。全エンドポイントが応答しなかった場合はエラーをスローする。

## 2. Room クラス

**ファイル:** `packages/sdk/src/Room.ts`

サーバー上の Room インスタンスへの接続を表す。状態の受信、メッセージの送受信、イベントの購読を行う。

### 型パラメータ

| パラメータ | 説明 |
|-----------|------|
| `T` | Room の型（サーバー側定義から推論） |
| `State` | Room の状態型 |

### コンストラクタ

```typescript
constructor(name: string, rootSchema?: SchemaConstructor<State>)
```

> 通常はユーザーが直接呼び出さない。`Client` のマッチメイキングメソッドが内部で使用する。

### プロパティ

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `roomId` | `string` | Room の一意識別子 |
| `sessionId` | `string` | このクライアントのセッション ID |
| `reconnectionToken` | `string` | 再接続用トークン（形式: `roomId:token`） |
| `name` | `string` | Room 名 |
| `connection` | `Connection` | WebSocket/WebTransport 接続 |
| `state` | `State` (getter) | Room の現在の状態（読み取り専用） |
| `serializerId` | `string` | 使用中のシリアライザ ID |
| `serializer` | `InferSerializer<State>` | シリアライザインスタンス |
| `reconnection` | `ReconnectionOptions` | 自動再接続設定 |

---

### シグナル（イベント）

Room クラスのイベントは Signal パターンで実装されている。各シグナルに対してコールバックを登録できる。

| シグナル | コールバック型 | 説明 |
|---------|--------------|------|
| `onStateChange` | `(state: State) => void` | 状態が変更されたとき（初回の完全同期・パッチ適用時の両方） |
| `onError` | `(code: number, message?: string) => void` | エラー発生時 |
| `onLeave` | `(code: number, reason?: string) => void` | Room から退出したとき（正常退出・強制退出の両方） |
| `onReconnect` | `() => void` | 自動再接続が成功したとき |
| `onDrop` | `(code: number, reason?: string) => void` | 接続が切断されたとき（再接続試行前に発火） |

**Signal API:**

```typescript
// コールバック登録
room.onStateChange((state) => { /* ... */ });

// 一度だけ実行
room.onStateChange.once((state) => { /* ... */ });

// 解除
const unbind = room.onStateChange((state) => { /* ... */ });
unbind(); // 登録解除

// 全解除
room.onStateChange.clear();
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `type` | `string \| number` | メッセージの種別 |

#### `onMessage()`

サーバーからのメッセージを受信するハンドラを登録する。

```typescript
// 特定のメッセージタイプを購読
public onMessage<Payload = any>(
    type: string | number,
    callback: (payload: Payload) => void
): () => void

// 全メッセージを購読（ワイルドカード）
public onMessage<Payload = any>(
    type: "*",
    callback: (messageType: string | number, payload: Payload) => void
): () => void
```

| 引数 | 型 | 説明 |
|------|-----|------|
| `type` | `string \| number \| "*"` | メッセージの種別。`"*"` で全メッセージを購読 |
| `callback` | `Function` | 受信時のコールバック。`"*"` の場合は第1引数にメッセージタイプが渡される |

**戻り値:** `() => void` -- 登録解除関数

> **注意:** 登録されていないメッセージタイプを受信すると、コンソールに警告が表示される。`"*"` ハンドラが登録されている場合は、未登録タイプもそちらにフォールバックする。

---

#### `leave()`

| 引数 | 型 | デフォルト | 説明 |
|------|-----|-----------|------|
| `consented` | `boolean` | `true` | `true`: サーバーに退出を通知してから切断。`false`: 即座に接続を切断 |

**戻り値:** `Promise<number>` -- クローズコード

---

#### `ping()`

| 引数 | 型 | 説明 |

| `callback` | `(ms: number) => void` | 往復時間（ミリ秒）を受け取るコールバック |

```typescript

```

### 自動再接続（ReconnectionOptions）

Room は接続切断時に自動再接続を試行する。`room.reconnection` プロパティで設定を変更できる。

```typescript
interface ReconnectionOptions {
    enabled: boolean;              // 自動再接続の有効/無効（デフォルト: true）
    maxRetries: number;            // 最大リトライ回数（デフォルト: 15）
    delay: number;                 // 初期ディレイ（デフォルト: 100ms）
    minDelay: number;              // 最小ディレイ（デフォルト: 100ms）
    maxDelay: number;              // 最大ディレイ（デフォルト: 5000ms）
    minUptime: number;             // 再接続試行の最小稼働時間（デフォルト: 5000ms）
    retryCount: number;            // 現在のリトライ回数（読み取り専用）
    backoff: (attempt: number, delay: number) => number;  // バックオフ関数
    maxEnqueuedMessages: number;   // バッファリングメッセージの最大数（デフォルト: 10）
    enqueuedMessages: Array<{ data: Uint8Array }>;  // バッファ中のメッセージ
    isReconnecting: boolean;       // 再接続中フラグ（読み取り専用）
}
```

**再接続のフロー:**

1. 接続切断時、`onDrop` シグナルが発火する
2. `minUptime` 条件を満たしている場合、指数バックオフで再接続を試行する
3. 再接続成功時は `onReconnect` シグナルが発火し、バッファ中のメッセージが送信される
4. `maxRetries` 回失敗した場合は `onLeave` が `CloseCode.FAILED_TO_RECONNECT` で発火する

**再接続が発生するクローズコード:**

- `CloseCode.NO_STATUS_RECEIVED`
- `CloseCode.ABNORMAL_CLOSURE`
- `CloseCode.GOING_AWAY`
- `CloseCode.MAY_TRY_RECONNECT`

### FetchRequestOptions

```typescript
type FetchRequestOptions = Omit<RequestInit, "body"> & {
    headers?: CommonHeaders | Headers | HeadersInit;
    body?: any;                            // JSON シリアライズ可能なオブジェクト
    query?: Record<string, any>;           // クエリパラメータ
    params?: Record<string, any> | string[];  // URL パスパラメータ（:id → { id: "xxx" }）
}
```

**動作の詳細:**

- `body` が JSON シリアライズ可能なオブジェクトの場合、自動的に `JSON.stringify` される
- `body` 内の `Date` オブジェクトは ISO 文字列に変換される
- `Content-Type` ヘッダーが未設定の場合、JSON ボディでは `application/json` が自動設定される
- `authToken` が設定済みの場合、`Authorization: Bearer <token>` が自動付与される
- `credentials` はデフォルトで `"include"`
- `params` は URL のパスパラメータ（`:id` 等）を置換する
- レスポンスが非 OK（4xx/5xx）の場合は `ServerError` をスローする
- ネットワークエラー時も `ServerError` をスローする
