## 2. 主要クラスの関係

### クラス階層図

```
Server
  ├── transport: Transport          # WebSocket 等の接続層
  ├── presence: Presence            # プロセス間通信（デフォルト: LocalPresence）
  ├── driver: MatchMakerDriver      # Room キャッシュ永続化（デフォルト: LocalDriver）
  └── router: Router                # HTTP ルーティング（better-call ベース）
       │
       ▼
MatchMaker (モジュール)
  ├── handlers: { [name]: RegisteredHandler }  # define() で登録された Room 定義
  ├── rooms: { [roomId]: Room }                # 実行中の Room インスタンス
  ├── presence / driver                        # Server から注入
  └── joinOrCreate / create / join / reconnect # マッチメイキング操作
       │
       ▼
Room<T>
  ├── clients: ClientArray<Client>   # 接続中のクライアント一覧
  ├── state: T                       # @colyseus/schema による同期ステート
  ├── presence: Presence             # スコープ付き Presence（Room 破棄時に自動クリーンアップ）
  ├── clock: Clock                   # タイマー管理
  ├── _serializer: Serializer        # 状態シリアライズ戦略
  └── ライフサイクルメソッド
       ├── onCreate(options)
       ├── onAuth(client, options)
       ├── onJoin(client, options, auth)
       ├── onMessage(type, callback)   # messages プロパティでも宣言可能
       ├── onDrop(client, code)        # 予期しない切断
       ├── onReconnect(client)
       ├── onLeave(client, code)
       └── onDispose()
       │
       ▼
Client (インターフェース)
  ├── sessionId: string              # セッション固有 ID
  ├── state: ClientState             # JOINING → JOINED → LEAVING → CLOSED
  ├── auth?: any                     # onAuth の戻り値
  ├── userData?: any                 # サーバー側カスタムデータ
  ├── view?: StateView               # @view() デコレータ使用時の個別ビュー
  ├── reconnectionToken: string      # 再接続用トークン
  ├── send(type, message)            # 型付きメッセージ送信
  ├── sendBytes(type, bytes)         # バイナリ送信
  └── leave(code)                    # 切断
```

### Server の初期化コード（ソース抜粋）

```typescript
// packages/core/src/Server.ts
constructor(options: ServerOptions = {}) {
    this.presence = options.presence || new LocalPresence();
    this.driver = options.driver || new LocalDriver();
    this.attach(options);

    matchMaker.setup(
      this.presence,
      this.driver,
      options.publicAddress,
      options.selectProcessIdToCreateRoom,
    );
}
```

`Server` はコンストラクタで `Presence` と `Driver` をデフォルト（ローカル実装）またはオプション指定で初期化し、`MatchMaker` モジュールに注入する。

### defineServer ヘルパー（0.17.x の推奨パターン）

```typescript
// packages/core/src/Server.ts
export function defineServer<T, R>(options: DefineServerOptions<T, R>): Server<T, R> {
  const { rooms, routes, ...serverOptions } = options;
  const server = new Server<T, R>(serverOptions);
  server.router = routes;
  for (const [name, handler] of Object.entries(rooms)) {
    handler.name = name;
    matchMaker.addRoomType(handler);
  }
  return server;
}
```

### 3.2 マッチメイキング詳細

`MatchMaker` モジュールは 4 つの主要操作を提供する:

| 操作 | 説明 |
|------|------|
| `joinOrCreate(name, options)` | 既存の空き Room を探し、なければ新規作成。最大 5 回リトライ |
| `create(name, options)` | 常に新しい Room を作成 |
| `join(name, options)` | 既存 Room にのみ参加（なければエラー） |
| `reconnect(roomId, options)` | `reconnectionToken` で既存セッションに再接続 |

**同時作成の排他制御:**

```typescript
// packages/core/src/MatchMaker.ts — joinOrCreate 内
await concurrentJoinOrCreateRoomLock(handler, concurrencyKey, async (roomId?) => {
    // 同じ filterOptions を持つ Room が並行して複数作成されるのを防止
    if (!room) {
        room = await createRoom(roomName, clientOptions);
        presence.publish(`concurrent:${handler.name}:${concurrencyKey}`, room.roomId);
    }
    return room;
});
```

`filterOptions` に基づくロックキーで、同一条件の Room が重複作成されないよう `Presence` の pub/sub を活用する。

### 3.3 状態同期

Room の状態同期は **サーバー権威モデル** で動作する:

1. **状態の設定:** `Room.state` に `@colyseus/schema` オブジェクトを代入すると、自動的に `SchemaSerializer` が選択される
2. **パッチレート:** デフォルト 20fps (50ms) で `broadcastPatch()` が実行される
3. **差分計算:** `Serializer.applyPatches()` が変更部分のみをバイナリエンコード
4. **クライアントビュー:** `@view()` デコレータ使用時は `Client.view` (StateView) を通じてクライアントごとに異なるステートを配信

```typescript
// packages/core/src/Room.ts — パッチレート設定
public patchRate: number | null = DEFAULT_PATCH_RATE; // 1000 / 20 = 50ms

// state setter が SchemaSerializer を自動選択
set: (newState) => {
    if (newState?.constructor[Symbol.metadata] !== undefined || newState[$changes] !== undefined) {
        this.setSerializer(new SchemaSerializer());
    }
    this._serializer.reset(newState);
}
```

### 3.4 メッセージング

クライアントからのメッセージは `messages` プロパティまたは `onMessage()` で宣言的に処理される:

```typescript
// Room クラス内（0.17.x 推奨パターン）
messages = {
    move: validate(z.object({ x: z.number(), y: z.number() }), (client, message) => {
        // Standard Schema V1 互換のバリデーション付き

    }),

        // フォールバックハンドラ（"*" に変換される）
    }

```

**レート制限:** `maxMessagesPerSecond` でクライアントあたりの送信制限が可能。超過時は切断される。

## 4. プラガブルアーキテクチャ

Colyseus は 4 つの拡張ポイントを持つプラグイン構造を採用している。すべて `ServerOptions` で差し替え可能。

| 実装 | パッケージ | 用途 |
|------|-----------|------|

### 4.3 Presence（プロセス間通信）

```typescript
// packages/core/src/presence/Presence.ts
export interface Presence {
    subscribe(topic, callback): Promise<this>;
    unsubscribe(topic, callback?);
    publish(topic, data);
    // Key-Value 操作
    set(key, value); get(key); del(key); setex(key, value, seconds);
    // Set 操作
    sadd(key, value); smembers(key); srem(key, value); sinter(...keys);
    // Hash 操作
    hset(key, field, value); hget(key, field); hgetall(key); hdel(key, field);
    // List 操作
    rpush(key, ...values); lpush(key, ...values); rpop(key); lpop(key);
    // Atomic 操作
    incr(key); decr(key); hincrby(key, field, value);
    shutdown(): void;
}
```

| 実装 | パッケージ | 用途 |
|------|-----------|------|
| LocalPresence | `@colyseus/core` (内蔵) | デフォルト。単一プロセス用。EventEmitter + Map |
| RedisPresence | `@colyseus/redis-presence` | マルチプロセス/マルチサーバー。Redis Pub/Sub |

Presence は以下で使用される:
- **IPC (プロセス間通信):** リモートプロセスへの Room 作成リクエスト、ヘルスチェック
- **同時作成ロック:** `concurrentJoinOrCreateRoomLock` での排他制御
- **Room スコープ:** `createScopedPresence()` で Room ごとのサブスクリプションを管理し、Room 破棄時に自動解除
