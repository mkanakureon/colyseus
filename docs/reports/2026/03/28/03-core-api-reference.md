## 1. Room クラス

**ファイル:** `packages/core/src/Room.ts`

ゲームセッションを実装し、クライアントグループ間の通信チャネルとして機能する中心クラス。マッチメイキング時にオンデマンドで生成される。

### 1.1 型パラメータ

```typescript
class Room<T extends RoomOptions = RoomOptions>
```

`RoomOptions` は以下のプロパティを持つ:

| プロパティ | 型 | 説明 |
|-----------|------|------|
| `state` | `object` | ルームの状態オブジェクト（Schema） |
| `metadata` | `any` | マッチメイキング用メタデータ |
| `client` | `Client` | クライアント型の指定 |

使用例:

```typescript
class MyRoom extends Room<{
  state: MyState,
  metadata: { difficulty: string },
  client: MyClient
}> { ... }
```

### 1.2 プロパティ

| プロパティ | 型 | デフォルト | 説明 |
|-----------|------|-----------|------|
| `roomId` | `string` | 自動生成（9文字） | ルームの一意 ID。`onCreate()` 中のみ上書き可 |
| `roomName` | `string` | - | `define()` で指定した名前。上書き不可 |
| `state` | `ExtractRoomState<T>` | - | Schema 状態オブジェクト。代入時にシリアライザが自動設定される |
| `metadata` | `ExtractRoomMetadata<T>` | - | マッチメイキング用メタデータ。`onCreate()` 中のみ直接代入可。以降は `setMatchmaking()` を使用 |
| `clients` | `ClientArray<Client>` | `[]` | 接続中のクライアント配列 |
| `clock` | `Clock` | - | タイマーイベント管理。ルーム dispose 時にクリアされる |
| `maxClients` | `number` | `Infinity` | 最大クライアント数。到達時に自動ロック |
| `autoDispose` | `boolean` | `true` | 最後のクライアント切断時に自動 dispose するか |
| `patchRate` | `number \| null` | `50`（20fps） | 状態ブロードキャストの頻度（ミリ秒）。`null` / `0` で無効化 |
| `maxMessagesPerSecond` | `number` | `Infinity` | クライアントの毎秒最大メッセージ数。超過で切断 |
| `seatReservationTimeout` | `number` | `15` | 座席予約のタイムアウト（秒）。環境変数 `COLYSEUS_SEAT_RESERVATION_TIME` でグローバル設定可 |
| `locked` | `boolean`（readonly） | `false` | ルームのロック状態 |
| `presence` | `Presence` | - | Presence API インスタンス |

### 1.3 ライフサイクルメソッド

#### onCreate

```typescript
public onCreate?(options: any): void | Promise<any>
```

ルーム作成時に呼ばれる。`options` はマッチメイキング時のクライアントオプションとデフォルトオプションのマージ。

#### onAuth

```typescript
public onAuth(
  client: Client,
  options: any,
  context: AuthContext
): any | Promise<any>
```

`onJoin()` の前に呼ばれる認証メソッド。デフォルトは `true` を返す。`false` や例外で認証失敗。`client.auth` に結果が格納される。

静的メソッド版も存在（推奨）:

```typescript
static async onAuth(
  token: string,
  options: any,
  context: AuthContext
): Promise<unknown>
```

`AuthContext` の型:

```typescript
type AuthContext = {
  token?: string;
  headers: Headers;
  ip: string | string[];
  req?: any;
};
```

#### onJoin

```typescript
public onJoin?(
  client: ExtractRoomClient<T>,
  options?: any,
  auth?: any
): void | Promise<any>
```

クライアントがルームに参加したとき呼ばれる。`auth` は非推奨（`client.auth` を使用）。

#### onDrop

```typescript
public onDrop?(
  client: ExtractRoomClient<T>,
  code?: number
): void | Promise<any>
```

クライアントが**意図せず**切断したとき呼ばれる（ネットワーク断等）。`allowReconnection()` を呼び出して再接続を許可できる。定義されていない場合は `onLeave` が代わりに呼ばれる。

#### onReconnect

```typescript
public onReconnect?(
  client: ExtractRoomClient<T>
): void | Promise<any>
```

クライアントが再接続したとき呼ばれる。

#### onLeave

```typescript
public onLeave?(
  client: ExtractRoomClient<T>,
  code?: number
): void | Promise<any>
```

クライアントが**完全に**ルームを離脱したとき呼ばれる。`onDrop` が定義されている場合、意図的な離脱（`CloseCode.CONSENTED`）時のみ呼ばれる。`onDrop` 未定義時はすべての離脱で呼ばれる。

#### onDispose

```typescript
public onDispose?(): void | Promise<any>
```

ルームが破棄されるときに呼ばれる。

#### onBeforePatch

```typescript
public onBeforePatch?(state: ExtractRoomState<T>): void | Promise<any>
```

状態パッチがブロードキャストされる直前に呼ばれる。

#### onBeforeShutdown

```typescript
public onBeforeShutdown(): void
```

サーバーのグレースフルシャットダウン時に呼ばれる。デフォルト実装は `this.disconnect()` を呼ぶ。オーバーライド可。

#### onUncaughtException

```typescript
public onUncaughtException?(
  error: RoomException,
  methodName: RoomMethodName
): void
```

定義すると、すべてのライフサイクルフック（`onMessage`, `onAuth`, `onJoin`, `onLeave`, `onCreate`, `onDispose`, `setSimulationInterval`, `clock.setTimeout/setInterval`）が try/catch でラップされ、例外がこのメソッドに転送される。**実験的機能。**

#### onCacheRoom / onRestoreRoom（devMode 専用）

```typescript
public onCacheRoom?(): any
public onRestoreRoom?(cached?: any): void
```

`devMode` 有効時、シャットダウン時にルーム状態をキャッシュし、再起動時に復元する。

### 1.4 状態管理メソッド

#### setMatchmaking

```typescript
public async setMatchmaking(updates: {
  metadata?: ExtractRoomMetadata<T>;
  private?: boolean;
  locked?: boolean;
  maxClients?: number;
  unlisted?: boolean;
  [key: string]: any;
}): Promise<void>
```

複数のマッチメイキングプロパティを一度に更新する**推奨メソッド**。単一の persist 操作で効率的に更新される。

#### setMetadata

```typescript
public async setMetadata(
  meta: Partial<ExtractRoomMetadata<T>>,
  persist: boolean = true
): Promise<void>
```

ルームのメタデータを部分更新する。既存メタデータにマージされる。

#### setPrivate

```typescript
public async setPrivate(
  bool: boolean = true,
  persist: boolean = true
): Promise<void>
```

ルームをプライベートに設定する。プライベートルームはマッチメイキングの検索結果に表示されない。

### 1.5 クライアント管理メソッド

#### lock

```typescript
public async lock(): Promise<void>
```

ルームをロックし、新規クライアントの参加を防ぐ。

#### unlock

```typescript
public async unlock(): Promise<void>
```

ルームのロックを解除し、新規クライアントの参加を許可する（`maxClients` 未達の場合）。

#### disconnect

```typescript
public disconnect(closeCode: number = CloseCode.CONSENTED): Promise<any>
```

すべての接続クライアントを切断し、ルームを dispose する。`onCreate()` 中は呼び出し不可。

#### allowReconnection

```typescript
public allowReconnection(
  previousClient: Client,
  seconds: number | "manual"
): Deferred<Client>
```

`onDrop()` / `onLeave()` 内で使用し、指定秒数内の再接続を許可する。`"manual"` は無制限（`Infinity`）。返される `Deferred` の `.reject()` で強制キャンセル可。

#### hasReachedMaxClients

```typescript
public hasReachedMaxClients(): boolean
```

接続クライアント数 + 予約済み座席数が `maxClients` に達しているか判定。

#### hasReservedSeat

```typescript
public hasReservedSeat(
  sessionId: string,
  reconnectionToken?: string
): boolean
```

指定 `sessionId` に座席予約が存在するか確認。

### 1.6 メッセージングメソッド

#### onMessage

```typescript
// 特定メッセージタイプ
public onMessage<T = any>(
  messageType: string | number,
  callback: (client: Client, message: T) => void
): () => void

// バリデーション付き
public onMessage<T = any>(
  messageType: string | number,
  validationSchema: StandardSchemaV1<T>,
  callback: (client: Client, message: T) => void
): () => void

// ワイルドカード（全メッセージ）
public onMessage<T = any>(
  messageType: '*',
  callback: (client: Client, type: string | number, message: T) => void
): () => void
```

クライアントからのメッセージハンドラを登録する。戻り値の関数を呼ぶとハンドラ解除。`StandardSchemaV1`（Zod 等）によるバリデーションに対応。

#### messages プロパティ（宣言的メッセージハンドラ）

```typescript
public messages?: Messages<any>;
```

オブジェクト形式で宣言的にメッセージハンドラを定義できる。`_` キーはワイルドカードハンドラ。

```typescript
messages = {
  move: (client, message) => { ... },
  attack: validate(z.object({ target: z.string() }), (client, message) => { ... }),
  _: (client, type, message) => { ... }, // fallback
}
```

#### onMessageBytes

```typescript
public onMessageBytes<T = any>(
  messageType: string | number,
  callback: (client: Client, message: T) => void
): () => void
```

バイナリメッセージ用ハンドラ登録。

#### broadcast

```typescript
public broadcast<K extends keyof Client['~messages']>(
  type: K,
  ...args: MessageArgs<Client['~messages'][K], IBroadcastOptions>
): void
```

接続中の全クライアントにメッセージをブロードキャストする。

`IBroadcastOptions`:

```typescript
interface IBroadcastOptions extends ISendOptions {
  except?: Client | Client[];  // 除外するクライアント
  afterNextPatch?: boolean;     // 次のパッチ後に送信
}
```

#### broadcastBytes

```typescript
public broadcastBytes(
  type: string | number,
  message: Uint8Array,
  options: IBroadcastOptions
): void
```

バイナリデータのブロードキャスト。

#### broadcastPatch

```typescript
public broadcastPatch(): boolean
```

状態の変更を検知し、全クライアントにパッチをブロードキャストする。通常は `patchRate` に基づいて自動呼び出しされる。変更があった場合 `true` を返す。

### 1.7 シミュレーション・タイマー

#### setSimulationInterval

```typescript
public setSimulationInterval(
  onTickCallback?: SimulationCallback,
  delay: number = 16.66  // 60fps
): void
```

ゲームループを設定する。`SimulationCallback` は `(deltaTime: number) => void`。`clock.tick()` が自動的に呼ばれる。

#### clock

`Clock` インスタンス（`@colyseus/timer`）。以下のメソッドが利用可能:

- `clock.setTimeout(callback, delay)` - 遅延実行
- `clock.setInterval(callback, delay)` - 定期実行
- `clock.deltaTime` - 前回ティックからの経過時間
- `clock.elapsedTime` - ルーム開始からの経過時間
- `clock.currentTime` - 現在時刻

### 1.8 非推奨メソッド

| メソッド | 代替 |
|---------|------|
| `setState(newState)` | `this.state = newState` |
| `setPatchRate(ms)` | `this.patchRate = ms` |
| `setSeatReservationTime(seconds)` | `this.seatReservationTimeout = seconds` |
| `send(client, type, message)` | `client.send(type, message)` |

### 1.9 ヘルパー関数

#### validate

```typescript
function validate<T extends StandardSchemaV1>(
  format: T,
  handler: (client: Client, message: StandardSchemaV1.InferOutput<T>) => void
): MessageHandlerWithFormat<T>
```

`messages` プロパティ内でバリデーション付きハンドラを定義するヘルパー。

#### defineRoom

```typescript
function defineRoom<T extends Type<Room>>(
  roomKlass: T,
  defaultOptions?: Parameters<NonNullable<InstanceType<T>['onCreate']>>[0]
): RegisteredHandler<InstanceType<T>>
```

`defineServer` の `rooms` オプション内でルーム定義を作成するヘルパー。

## 2. Server クラス

**ファイル:** `packages/core/src/Server.ts`

Colyseus サーバーのエントリーポイント。Transport、Presence、Driver の初期化とルーム定義を管理する。

### 2.1 コンストラクタ

```typescript
constructor(options: ServerOptions = {})
```

`ServerOptions`:

| オプション | 型 | デフォルト | 説明 |
|-----------|------|-----------|------|
| `publicAddress` | `string` | - | サーバーのパブリックアドレス |
| `presence` | `Presence` | `LocalPresence` | Presence 実装（Redis 等） |
| `driver` | `MatchMakerDriver` | `LocalDriver` | マッチメイキングドライバ |
| `transport` | `Transport` | `WebSocketTransport` | トランスポート層 |
| `gracefullyShutdown` | `boolean` | `true` | グレースフルシャットダウン有効化 |
| `greet` | `boolean` | `true` | 起動時のグリーティングメッセージ表示 |
| `logger` | `any` | - | カスタムロガー |
| `beforeListen` | `() => Promise<void> \| void` | - | `listen()` 前に実行するコールバック（DB 接続等） |
| `express` | `(app: express.Application) => void` | - | Express ルート設定コールバック |
| `selectProcessIdToCreateRoom` | `SelectProcessIdCallback` | 最小ルーム数プロセス | ルーム作成プロセス選択関数 |
| `isStandaloneMatchMaker` | `boolean` | `false` | スタンドアロンマッチメーカーモード |
| `devMode` | `boolean` | `false` | 開発モード（ルーム復元等） |

### 2.2 メソッド

#### listen

```typescript
public async listen(
  port: number | string,
  hostname?: string,
  backlog?: number,
  listeningListener?: Function
): Promise<void>
```

指定ポートでサーバーを起動する。`beforeListen` コールバック実行 → マッチメーカー初期化 → Transport リスニング開始の順で処理。Unix ソケットパスも指定可。

#### define

```typescript
public define<T extends Type<Room>>(
  roomClass: T,
  defaultOptions?: OnCreateOptions<T>
): RegisteredHandler

public define<T extends Type<Room>>(
  name: string,
  roomClass: T,
  defaultOptions?: OnCreateOptions<T>
): RegisteredHandler
```

ルームタイプをマッチメイキングに登録する。`name` 省略時はクラス名が使用される。返される `RegisteredHandler` でフィルタやソートオプションを設定可能。

#### removeRoomType

```typescript
public removeRoomType(name: string): void
```

マッチメイキングからルーム定義を削除する。既存のルームインスタンスは破棄しない。

#### attach

```typescript
public async attach(options: ServerOptions): Promise<void>
```

Transport を初期化し、Express アプリケーションをセットアップする。

#### simulateLatency

```typescript
public simulateLatency(milliseconds: number): void
```

クライアント・サーバー間のレイテンシをシミュレートする（ラウンドトリップ時間）。`0` で無効化。開発・テスト用。

#### onShutdown

```typescript
public onShutdown(callback: () => void | Promise<any>): void
```

シャットダウン完了後に実行されるコールバックを登録。

#### onBeforeShutdown

```typescript
public onBeforeShutdown(callback: () => void | Promise<any>): void
```

シャットダウン開始前に実行されるコールバックを登録。

#### gracefullyShutdown

```typescript
public async gracefullyShutdown(
  exit: boolean = true,
  err?: Error
): Promise<void>
```

グレースフルシャットダウンを実行する。全ルームをロック → dispose → Transport/Presence/Driver をシャットダウン。

### 2.3 ヘルパー関数

#### defineServer

```typescript
function defineServer<T, R>(options: DefineServerOptions<T, R>): Server<T, R>
```

`DefineServerOptions`:

```typescript
type DefineServerOptions<T, R> = ServerOptions & {
  rooms: T;       // ルーム定義マップ
  routes?: R;     // カスタムルーター
};
```

サーバーとルーム定義を一括で設定するファクトリ関数。

## 3. MatchMaker モジュール

#### joinById

```typescript
  authContext?: AuthContext
```

特定の `roomId` のルームに参加する。ルームが見つからないかロック済みの場合はエラー。

### 3.5 シャットダウン・ヘルスチェック

#### gracefullyShutdown


#### マッチング処理フロー

1. `cycleTickInterval` 毎に `reassignMatchGroups()` が呼ばれる
2. 全クライアントをランク昇順でソート
3. `compare()` 関数（デフォルト: ランク差 < 10 または比率 <= 200%）でグループ分け
4. グループが `maxPlayers` に達したら `ready = true`
5. `maxWaitingCyclesForPriority` 超過で `highPriority` フラグ（ランク互換性を無視してグループ参加）
6. `maxWaitingCycles` 超過 + `allowIncompleteGroups` で不完全グループもマッチ成立
7. `ready` なグループに対して `onGroupReady()` → ルーム作成 → 座席予約 → クライアントに通知
