## 1. Transport インターフェースの解説

### 基底クラス: `Transport`

`packages/core/src/Transport.ts` に定義された抽象クラス。すべての Transport 実装はこのクラスを継承する。

```typescript
export abstract class Transport {
    public protocol?: string;
    public server?: http.Server | https.Server;

    public abstract listen(port?: number | string, hostname?: string,
                           backlog?: number, listeningListener?: Function): this;
    public abstract shutdown(): void;
    public abstract simulateLatency(milliseconds: number): void;

    // オプショナル
    public getExpressApp?(): Promise<Application> | Application | undefined;
    public bindRouter?(router: Router): void;
}
```

#### 必須メソッド

| メソッド | 役割 |
|---------|------|
| `listen(port, hostname, backlog, callback)` | 指定ポートでサーバーを起動する。メソッドチェーンのため `this` を返す |
| `shutdown()` | サーバーを停止し、全接続を閉じる |
| `simulateLatency(ms)` | デバッグ用。送信に人工的な遅延を挿入する |

#### オプショナルメソッド

| メソッド | 役割 |
|---------|------|
| `getExpressApp()` | Express 互換の HTTP ルーティング用アプリケーションを返す。uWebSockets の場合は `uwebsockets-express` モジュール経由 |
| `bindRouter(router)` | `@colyseus/better-call` の Router をバインドする。uWebSockets と Bun で独自実装あり |

#### プロパティ

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `protocol` | `string?` | トランスポートプロトコルの識別子。H3Transport では `"h3"` が設定される |
| `server` | `http.Server \| https.Server?` | 内部で使用する HTTP/HTTPS サーバーインスタンス |

### Client インターフェース

各 Transport は独自の Client 実装を持つ。すべて `Client` および `ClientPrivate` インターフェースを実装する。

主要メソッド:
- `send(type, message, options?)` -- MsgPack エンコードされたメッセージを送信
- `sendBytes(type, bytes, options?)` -- 生バイナリを送信
- `raw(data, options?, cb?)` -- 低レベルの直接送信
- `leave(code?, data?)` -- クライアントを切断
- `error(code, message?)` -- エラーコードをクライアントに送信
- `enqueueRaw(data, options?)` -- JOIN 完了前はキューに蓄積、完了後は即時送信

### グローバル Transport 管理

```typescript
setTransport(transport: Transport)  // Transport インスタンスをグローバルに登録
getTransport()                       // 現在の Transport を取得
```

`Server` がインスタンス化される際に `setTransport()` が呼ばれ、フレームワーク内部から参照される。

### 機能比較表

| 特徴 | ws-transport | uwebsockets-transport | bun-websockets | h3-transport | tcp-transport |
|------|:---:|:---:|:---:|:---:|:---:|
| **プロトコル** | WebSocket (RFC 6455) | WebSocket (RFC 6455) | WebSocket (RFC 6455) | HTTP/3 + WebTransport | TCP (raw) |
| **対応ランタイム** | Node.js | Node.js | Bun | Node.js | Node.js |
| **Express 互換** | ネイティブ | uwebsockets-express 経由 | bun-serve-express 経由 | ネイティブ (Express 直接) | なし |
| **Router バインド** | 不要 (Express 経由) | `bindRouter()` 独自実装 | `bindRouter()` 独自実装 | なし (Express 経由) | なし |
| **SSL/TLS** | Node.js https | uWebSockets SSLApp | Bun ネイティブ TLS | 必須 (HTTPS + QUIC) | なし |
| **Ping/Pong** | 手動実装 (configurable) | 自動 (`sendPingsAutomatically`) | Bun ランタイム管理 | セッション管理 | なし |
| **圧縮** | perMessageDeflate (デフォルト OFF) | uWebSockets DISABLED (デフォルト) | 設定可能 | N/A | なし |
| **maxPayload デフォルト** | 4KB | 4KB | 4KB | N/A | N/A |
| **Datagram サポート** | なし | なし | なし | あり (unreliable) | なし |
| **レイテンシシミュレーション** | あり | あり | あり | 未実装 | 未実装 |
| **本番推奨度** | 中 | 高 | 高 (Bun 環境) | 低 (実験的) | 使用不可 |

### パフォーマンス特性

| 指標 | ws-transport | uwebsockets-transport | bun-websockets |
|------|:---:|:---:|:---:|
| **メモリ効率** | 標準 | 高い (C++ バインディング) | 高い (Zig ベース) |
| **スループット** | 標準 | 高い (8-10x ws) | 高い |
| **接続管理** | EventEmitter ベース | WeakMap + ラッパー | WeakMap + ラッパー |
| **バックプレッシャー制御** | ws ライブラリ任せ | `maxBackpressure: 1MB` | Bun ランタイム管理 |
| **依存関係** | ws (pure JS) | uWebSockets.js (C++ native) | Bun 組み込み |

## 3. 各 Transport のセットアップ方法

### 3.1 WebSocketTransport (`@colyseus/ws-transport`)

最も標準的な Transport。Node.js の `ws` ライブラリベース。`defineServer()` を使う場合はデフォルトでこれが選択される。

#### インストール

```bash
npm install @colyseus/ws-transport
```

#### 基本セットアップ（Express 連携）

```typescript
import http from "http";
import express from "express";
import { Server } from "@colyseus/core";
import { WebSocketTransport } from "@colyseus/ws-transport";
import { MyRoom } from "./MyRoom";

const app = express();
const server = http.createServer(app);
const transport = new WebSocketTransport({ server });

const gameServer = new Server({ transport });

app.use(express.json());
app.get("/hello", (req, res) => {
  res.json({ hello: "world!" });
});

gameServer.define("my_room", MyRoom);

gameServer.listen(2567)
  .then(() => console.log("Listening on ws://localhost:2567"));
```

#### オプション (`TransportOptions`)

```typescript
new WebSocketTransport({
  server,                    // http.Server インスタンス（省略時は自動生成）
  maxPayload: 4 * 1024,     // 最大ペイロードサイズ (デフォルト: 4KB)
  perMessageDeflate: false,  // メッセージ圧縮 (デフォルト: false)
  pingInterval: 3000,        // Ping 送信間隔 ms (デフォルト: 3000)
  pingMaxRetries: 2,         // Ping 無応答で切断するまでのリトライ数 (デフォルト: 2)
});
```

#### defineServer() 利用時（最もシンプル）

```typescript
import { defineServer, defineRoom } from "@colyseus/core";
import { MyRoom } from "./MyRoom";

// transport を省略すると WebSocketTransport がデフォルトで使われる
const gameServer = defineServer({
  rooms: {
    my_room: defineRoom(MyRoom),
  },
});

gameServer.listen(2567);
```

### 3.2 uWebSocketsTransport (`@colyseus/uwebsockets-transport`)

C++ ネイティブの uWebSockets.js をベースとした高性能 Transport。本番環境で推奨。

#### インストール

```bash
npm install @colyseus/uwebsockets-transport
# Express 互換が必要な場合:
npm install uwebsockets-express   # Express v5 -> ^2.0.1, Express v4 -> ^1.4.1
```

#### 基本セットアップ

```typescript
import { Server } from "@colyseus/core";
import { uWebSocketsTransport } from "@colyseus/uwebsockets-transport";
import { MyRoom } from "./MyRoom";

const transport = new uWebSocketsTransport({
  /* TransportOptions (WebSocketBehavior のサブセット) */
});

const gameServer = new Server({ transport });
gameServer.define("my_room", MyRoom);
gameServer.listen(2567);
```

#### defineServer() 利用時

```typescript
import { defineServer, defineRoom } from "@colyseus/core";
import { uWebSocketsTransport } from "@colyseus/uwebsockets-transport";
import { MyRoom } from "./MyRoom";

const gameServer = defineServer({
  transport: new uWebSocketsTransport({}),
  rooms: {
    my_room: defineRoom(MyRoom),
  },
});

gameServer.listen(2567);
```

#### オプション (`TransportOptions`)

```typescript
new uWebSocketsTransport({
  maxPayloadLength: 4 * 1024,      // 最大ペイロード (デフォルト: 4KB)
  maxBackpressure: 1024 * 1024,    // バックプレッシャー上限 (デフォルト: 1MB)
  compression: uWebSockets.DISABLED, // 圧縮 (デフォルト: DISABLED)
  sendPingsAutomatically: true,     // 自動 Ping (デフォルト: true)
}, {
  // uWebSockets.AppOptions (SSL 設定等)
  cert_file_name: "cert.pem",  // 設定すると SSLApp になる
  key_file_name: "key.pem",
});
```

#### 注意事項

- `uWebSockets.js` の HTTP リクエストハンドリングは Node.js の `http` モジュールと異なる。Express ミドルウェアを使う場合は `uwebsockets-express` が必要
- `bindRouter()` が独自に HTTP ルーティング（CORS 含む）を処理する
- WebSocket の `upgrade` ハンドラ内では `req` が同期コールバック内でのみ有効（非同期で参照不可）

### 3.3 BunWebSockets (`@colyseus/bun-websockets`)

Bun ランタイムのネイティブ WebSocket を利用する Transport。

#### インストール

```bash
npm install @colyseus/bun-websockets
```

#### 基本セットアップ

```typescript
import { Server } from "@colyseus/core";
import { BunWebSockets } from "@colyseus/bun-websockets";
import { MyRoom } from "./MyRoom";

const transport = new BunWebSockets({
  /* TransportOptions */
});

const gameServer = new Server({ transport });
gameServer.define("my_room", MyRoom);
gameServer.listen(2567);
```

#### defineServer() 利用時

```typescript
import { defineServer, defineRoom } from "@colyseus/core";
import { BunWebSockets } from "@colyseus/bun-websockets";
import { MyRoom } from "./MyRoom";

const gameServer = defineServer({
  transport: new BunWebSockets({}),
  rooms: {
    my_room: defineRoom(MyRoom),
  },
});

gameServer.listen(2567);
```

#### オプション (`TransportOptions`)

```typescript
new BunWebSockets({
  maxPayloadLength: 4 * 1024,  // 最大ペイロード (デフォルト: 4KB)
  // Bun の WebSocketHandler オプションがそのまま使える
  // (message, open, drain, close, ping, pong は Transport が管理するため除外)
});
```

#### 注意事項

- Bun ランタイムでのみ動作する（Node.js では使用不可）
- `Bun.serve()` の `fetch` ハンドラ内で WebSocket upgrade と HTTP ルーティングを統合処理
- Express 互換は `bun-serve-express` パッケージ経由
- `bindRouter()` で `@colyseus/better-call` の Router を直接バインド可能

### 3.4 H3Transport (`@colyseus/h3-transport`)

HTTP/3 (QUIC) ベースの WebTransport を使用する実験的 Transport。信頼性のある双方向ストリームとデータグラム（unreliable）の両方をサポート。

#### インストール

```bash
npm install @colyseus/h3-transport
```

#### 基本セットアップ

```typescript
import http from "http";
import express from "express";
import { Server } from "@colyseus/core";
import { H3Transport } from "@colyseus/h3-transport";
import { MyRoom } from "./MyRoom";

const app = express();
const server = http.createServer(app);

app.use(express.json());

const transport = new H3Transport({ server, app });
const gameServer = new Server({ transport });

gameServer.define("my_room", MyRoom);

gameServer.listen(2567)
  .then(() => console.log("Listening on https://localhost:2567"));
```

#### オプション (`TransportOptions`)

```typescript
new H3Transport({
  app,                     // Express アプリケーション (必須)
  server,                  // http.Server (オプション)
  cert: "...",             // SSL 証明書 (省略時は自動生成、10日間有効)
  key: "...",              // 秘密鍵
  secret: "mysecret",     // QUIC secret
  localProxy: "localhost:3000", // 開発用フロントエンドプロキシ
});
```

#### 注意事項

- HTTPS が必須（証明書未指定時は自己署名証明書を自動生成）
- `protocol` プロパティが `"h3"` に設定される
- `sendDatagram()` メソッドで unreliable なデータグラム送信が可能（ゲームの位置更新等に有用）
- WebTransport はブラウザサポートが限定的（Chrome 系のみ、2026年3月時点）
- `simulateLatency()` は未実装
- `@fails-components/webtransport` ライブラリに依存

### 3.5 TCPTransport (`@colyseus/tcp-transport`)

TCP ソケットベースの Transport。**現在動作しない（未完成）**。

```typescript
// ソースコード内のコメントより:
```

## 4. カスタム Transport の作り方

### 4.1 最小実装

```typescript
import http from "http";
import { Transport, matchMaker, connectClientToRoom, type Client, type ClientPrivate, ClientState, type ISendOptions, getMessageBytes, Protocol } from "@colyseus/core";
import { EventEmitter } from "events";

// Step 1: Client 実装を作る
class MyClient implements Client, ClientPrivate {
  '~messages': any;

  public id: string;
  public ref: EventEmitter = new EventEmitter();
  public sessionId: string;
  public state: ClientState = ClientState.JOINING;
  public reconnectionToken: string;
  public readyState: number = 1; // OPEN

  public _enqueuedMessages: any[] = [];
  public _afterNextPatchQueue: any[];
  public _joinedAt: number;

  constructor(sessionId: string) {
    this.id = this.sessionId = sessionId;
  }

  send(messageOrType: any, messageOrOptions?: any, options?: ISendOptions) {
    this.enqueueRaw(
      getMessageBytes.raw(Protocol.ROOM_DATA, messageOrType, messageOrOptions),
      options,
    );
  }

  sendBytes(type: string | number, bytes: Buffer | Uint8Array, options?: ISendOptions) {
    this.enqueueRaw(
      getMessageBytes.raw(Protocol.ROOM_DATA_BYTES, type, undefined, bytes),
      options,
    );
  }

  enqueueRaw(data: Uint8Array | Buffer, options?: ISendOptions) {
    if (options?.afterNextPatch) {
      this._afterNextPatchQueue.push([this, [data]]);
      return;
    }
    if (this.state !== ClientState.JOINED) {
      this._enqueuedMessages?.push(data);
      return;
    }
    this.raw(data, options);
  }

  raw(data: Uint8Array | Buffer, options?: ISendOptions, cb?: (err?: Error) => void) {
    // ここに実際の送信ロジックを実装
    // 例: this.socket.send(data);
  }

  error(code: number, message: string = '', cb?: (err?: Error) => void) {
    this.raw(getMessageBytes[Protocol.ERROR](code, message), undefined, cb);
  }

  leave(code?: number, data?: string) {
    // 接続を閉じる処理
  }

  close(code?: number, data?: string) {
    this.leave(code, data);
  }
}

// Step 2: Transport 実装を作る
class MyTransport extends Transport {
  public server: http.Server;

  constructor() {
    super();
    this.server = http.createServer();
  }

  listen(port: number, hostname?: string, backlog?: number, listeningListener?: () => void) {
    this.server.listen(port, hostname, backlog, listeningListener);
    // ここで接続受付を開始
    return this;
  }

  shutdown() {
    this.server.close();
  }

  simulateLatency(milliseconds: number) {
    // 開発用レイテンシシミュレーション
  }

  // オプション: Express 互換
  getExpressApp() {
    // Express アプリケーションを返す
    return undefined;
  }

  // オプション: Router バインド
  bindRouter(router: any) {
    // HTTP ルーティングを設定
  }

  // 接続時の処理
  protected async onConnection(sessionId: string, roomId: string, /* raw connection */) {
    const room = matchMaker.getLocalRoomById(roomId);
    const client = new MyClient(sessionId);

    try {
      await connectClientToRoom(room, client, {
        headers: new Headers(),
        token: undefined,
        ip: "127.0.0.1",
      }, {
        reconnectionToken: undefined,
        skipHandshake: false,
      });
    } catch (e: any) {
      client.error(e.code, e.message);
      // 接続を閉じる
    }
  }
}
```

### 4.2 実装チェックリスト

1. **`Transport` 抽象クラスを継承する**
2. **`Client` + `ClientPrivate` インターフェースを実装した Client クラスを作る**
3. **必須メソッドを実装する**: `listen()`, `shutdown()`, `simulateLatency()`
4. **接続ハンドラで `connectClientToRoom()` を呼ぶ** -- これが Room への参加処理を行う
5. **Ping/Pong を維持する** -- `sessionId` なしの接続は ping-pong ユーティリティとして扱う
6. **エラーハンドリング** -- 個別クライアントのエラーがサーバー全体をクラッシュさせないようにする
7. **Express 互換が必要なら `getExpressApp()` を実装する**
8. **`@colyseus/better-call` Router をサポートするなら `bindRouter()` を実装する**

### 4.3 既存実装から学ぶパターン

全 Transport 実装に共通するパターン:

- **Ping/Pong ユーティリティ**: `sessionId` も `roomId` もない接続は、1秒後に自動切断。メッセージ受信時に `Protocol.PING` を返す
- **メッセージエンキュー**: `ClientState.JOINED` 前のメッセージは `_enqueuedMessages` に蓄積される
- **`afterNextPatch` オプション**: 次の状態パッチ送信後にメッセージを送る仕組み
- **WeakMap による Client ラッパー**: uWebSockets と Bun では、raw WebSocket と Client ラッパーの紐付けに WeakMap を使用
- **レイテンシシミュレーション**: プロトタイプの `raw()` メソッドを `setTimeout` でラップする手法

## 5. 選定フローチャート

```
開始
  |
  v
ランタイムは何か？
  |
  +-- Bun --> @colyseus/bun-websockets
  |             (Bun ネイティブ WebSocket、高性能)
  |
  +-- Node.js
        |
        v
      本番環境か？
        |
        +-- YES --> パフォーマンス要件は？
        |             |
        |             +-- 高い (同時接続 1000+) --> @colyseus/uwebsockets-transport
        |             |     (C++ ネイティブ、最高スループット)
        |             |
        |             +-- 標準的 --> @colyseus/uwebsockets-transport
        |                   (安定性とパフォーマンスのバランス)
        |
        +-- NO (開発/プロトタイプ)
              |
              v
            Express ミドルウェアを多用するか？
              |
              +-- YES --> @colyseus/ws-transport
              |             (Express ネイティブ互換、セットアップ最小)
              |
              +-- NO --> @colyseus/ws-transport
                           (デフォルト、最もシンプル)

特殊要件:
  |
  +-- unreliable データグラムが必要 --> @colyseus/h3-transport
  |     (WebTransport、実験的、要 HTTPS)
  |
  +-- GameMaker 連携 --> @colyseus/tcp-transport
        (未完成、コントリビューション歓迎)
```

### 選定基準まとめ

| 状況 | 推奨 Transport | 理由 |
|------|---------------|------|
| 開発・プロトタイプ | `ws-transport` | セットアップが最もシンプル。`defineServer()` のデフォルト |
| Node.js 本番環境 | `uwebsockets-transport` | C++ ベースで最高のパフォーマンス。自動 Ping、バックプレッシャー制御 |
| Bun ランタイム | `bun-websockets` | Bun ネイティブ。追加依存なし |
| 低レイテンシ・unreliable 通信 | `h3-transport` | WebTransport のデータグラムが使える。ただし実験的 |
| `defineServer()` 利用時 | 省略可 (ws-transport) | `transport` オプション未指定でデフォルト選択される |

### 移行の容易さ

Transport はサーバー初期化時に1行変更するだけで切り替え可能。Room のコード変更は不要。

```typescript
// Before: ws-transport
const gameServer = new Server({ transport: new WebSocketTransport({ server }) });

// After: uwebsockets-transport (Room コードは変更なし)
const gameServer = new Server({ transport: new uWebSocketsTransport({}) });
```

唯一の注意点は Express ミドルウェアの互換性。uWebSockets や Bun では互換レイヤー（`uwebsockets-express`, `bun-serve-express`）が必要になる場合がある。
