# レポート保存 Skill

作業の振り返り・設計文書・調査結果などを `docs/reports/` に保存する。

## 保存先ルール

```
docs/reports/{年}/{月}/{日}/{連番}-{タイトル}.md
```

例: `docs/reports/2026/03/28/06-transport-benchmark-report.md`

- **年月日**: 今日の日付を使う（`YYYY/MM/DD`、ゼロ埋め）
- **連番**: その日のフォルダ内の既存ファイル数 + 1（ゼロ埋め2桁）
- **タイトル**: 英語ケバブケース、内容を簡潔に表す

### 種別サフィックス

| 種別 | サフィックス | 例 |
|---|---|---|
| 計画書 | `-plan` | `01-documentation-plan.md` |
| 実装レポート | `-report` | `06-room-lifecycle-report.md` |
| 設計書・仕様書 | `-spec` / `-design` | `02-transport-api-spec.md` |
| リファレンス | `-reference` | `03-core-api-reference.md` |
| ガイド | `-guide` | `07-transport-guide.md` |
| ベンチマーク | `-benchmark` | `04-ws-vs-uws-benchmark.md` |
| 調査・分析 | `-analysis` | `05-schema-performance-analysis.md` |

## 手順

1. 今日のフォルダを確認: `ls docs/reports/YYYY/MM/DD/`
2. 既存ファイル数から次の連番を決定
3. フォルダがなければ作成: `mkdir -p docs/reports/YYYY/MM/DD`
4. ファイルを作成

## デフォルト保存先のルール

**保存先が明示されていない場合、常に `docs/reports/` を使う。**

- 「ここに保存して」→ `docs/reports/` に保存
- 「文書を作って」「まとめて」など、保存先の言及がない場合も `docs/reports/` に保存

## トリガー

| ユーザーの表現 | 動作 |
|---|---|
| "レポート書いて" / "save report" | 直前の作業をレポートにまとめて保存 |
| "設計文書を保存" / "save spec" | 設計内容をまとめて保存 |
| "まとめを書いて" / "summarize" | 作業のまとめを保存 |
| "保存して" / "save" | 直前に議論した内容を保存 |
| "ドキュメント保存" | 同上 |

### 重要: 暗黙のトリガー

ユーザーが「保存」「文書」「ドキュメント」「まとめ」「report」「save」などのキーワードを含む指示をした場合、保存先が明示されていなくても `docs/reports/` を保存先とみなして発動する。

## レポートのフォーマット（必須）

すべてのドキュメントに**ヘッダーとフッター**を付ける。生成 AI がコンテキストとして取り込む際に、出典・信頼度・対象リポジトリを即座に判断できるようにするため。

### ヘッダー（必須）

```markdown
---
title: {ドキュメントタイトル}
type: {report | spec | reference | guide | analysis | benchmark | plan}
project: colyseus
version: 0.17.x
date: {YYYY-MM-DD}
author: Claude {モデル名}
---

# タイトル

> **Repository:** colyseus (authoritative multiplayer framework for Node.js)
> **Generated by:** Claude {現在のモデル名} | **Date:** {YYYY-MM-DD}
> **Scope:** {対象パッケージ or 全体}
```

### フッター（必須）

```markdown
---

## メタ情報

| 項目 | 値 |
|------|-----|
| 生成モデル | Claude {現在のモデル名} |
| 生成日 | {YYYY-MM-DD} |
| リポジトリ | colyseus (github.com/colyseus/colyseus) |
| バージョン | 0.17.x |
| 検証状態 | ソースコード読解に基づく / 未検証 |

> この文書は AI によって生成されました。内容の正確性はソースコードとの照合で確認してください。
```

### 本文テンプレート

```markdown
## 背景
（なぜこの作業が必要だったか）

## 方針
（どういうアプローチを取ったか）

## 変更ファイル一覧
| # | ファイル | 変更 |
|---|---------|------|
| 1 | `path/to/file` | 変更内容 |

## 実装の詳細
（コード例や設計の説明）

## テスト結果
（テストを実行した場合）

## 教訓
（今後に活かせるポイント）
```

- セクションは内容に応じて取捨選択する（全部必須ではない）
- コード例は実際のソースコードから抜粋
- 本文は日本語 or 英語（コンテキストに合わせる）
- **ヘッダーとフッターは必ず付ける**
